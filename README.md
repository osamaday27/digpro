# Digital Products Master Guide

دليل تفاعلي (Web App) بتصميم أبيض/فاتح افتراضي، مقسّم لملفات منفصلة عشان يكون سهل الفهم والتعديل. الصفحة دي **static** بالكامل — من غير سيرفر أو باك إند، وكل حاجة شغالة بمتصفح المستخدم.

---

## 1. بنية المشروع (Project Structure)

```
project/
├── index.html          ← الهيكل والمحتوى (HTML فقط، بدون تنسيق أو منطق)
├── css/
│   └── style.css        ← كل التنسيقات، بما فيها الوضع الفاتح والداكن
├── js/
│   └── app.js            ← كل المنطق: البيانات، التخزين، العرض، البحث...
└── README.md              ← الملف ده
```

**ليه اتقسّمت كده؟**
- `index.html` بقى نضيف وسهل تتابعه، لأنه مفيهوش أسطر CSS أو JS طويلة وسطه.
- `style.css` تقدر تفتحه لوحده وتغيّر أي لون أو مقاس من غير ما تلمس المنطق.
- `app.js` فيه كل "العقل" بتاع الصفحة: البيانات، وإزاي بتتحفظ، وإزاي الصفحات بتتنقل.

لو حبيت تستضيف الموقع، ارفع الفولدر `project/` كله زي ما هو على أي استضافة static (Netlify، Vercel، GitHub Pages، أو حتى استضافة عادية) — مفيهوش أي إعدادات سيرفر مطلوبة.

---

## 2. الوضع الفاتح والداكن (Light / Dark Mode)

- **الافتراضي دلوقتي هو الوضع الأبيض/الفاتح.** كل الألوان الأساسية متعرّفة في `:root` في أول `style.css`.
- لو المستخدم دوس على زرار القمر 🌙 فوق (`#darkToggle`)، بتتضاف كلاس `dark` على `<body>`، وده بيفعّل مجموعة الألوان التانية المعرّفة تحت `body.dark { ... }` في نفس الملف.
- الاختيار (فاتح/داكن) بيتحفظ في المتصفح، فلو المستخدم قفل الصفحة وفتحها تاني هتفضل زي ما سابها.

**عايز تغيّر الألوان؟** كل اللي عليك تعدّل القيم جوه `:root` (للفاتح) أو `body.dark` (للداكن) في `css/style.css`. كل عناصر الصفحة بتستخدم المتغيرات دي (`var(--accent)` مثلاً)، فتغيير مكان واحد بيغيّر الموقع كله.

## 3. شرح كل قسم في الصفحة

### Sidebar (القائمة الجانبية)
موجودة في `index.html` كـ `<aside class="sidebar">`. فيها كل روابط التنقل بين الصفحات، ومقسّمة لمجموعات (عام / المحتوى / أنت). في آخرها شريط بيوضح نسبة تقدمك العامة.

### Top Navbar
شريط علوي ثابت فيه:
- **Search** — مربع بحث بيدور في كل المحتوى لحظيًا.
- **Progress chip** — نسبة إنجازك الحالية.
- **Dark mode toggle** — زرار التبديل بين الفاتح والداكن.

### Dashboard
أول صفحة بتفتح. فيها:
- **Ring دائري** بيوضح نسبة إنجازك الكلية.
- **4 كروت إحصائيات** (Checklist المكتمل، مراحل Roadmap المكتملة، عدد المحفوظات، عدد الملاحظات).
- **"كمّل من هنا"** — روابط سريعة لأهم الحاجات اللي ممكن تعملها تالي.

### أنواع المنتجات (Products)
شبكة فيها 10 كروت (SaaS, AI Products, APIs, Laravel Templates... إلخ)، كل كارت فيه زرار نجمة لحفظه في المحفوظات.

### المسار التجاري (Business Track)
4 خطوات مرتّبة فعليًا (تسعير → نشر → تسويق → أول عميل) — مش عشوائية، عشان كده معمولة كـ "track" مرقّم بدل ما تكون كروت منفصلة.

### Roadmap (خطة الـ 90 يوم)
3 مراحل زمنية (1-30 / 31-60 / 61-90)، كل مرحلة فيها زرار "علّمها كمخلّصة" بيأثر على نسبة التقدم العامة.

### Checklist للتنفيذ
9 عناصر مقسّمة على 3 مجموعات (مطابقة لمراحل الـ Roadmap)، كل تشييك بيتحفظ فورًا.

### أدوات ومراجع (Resources)
6 فئات من الأدوات، كل واحدة قابلة للحفظ زي المنتجات بالظبط.

### ملاحظاتي (Notes)
تكتب ملاحظة حرة في مربع نص وتضغط حفظ، بتتخزن بتاريخها وتقدر تمسحها في أي وقت.

### المحفوظات (Bookmarks)
كل حاجة حفظتها (منتجات أو أدوات) بتتجمع هنا في مكان واحد.

### تحميلات (Downloads)
4 أزرار بتولّد ملفات `.txt` **حقيقية** من نفس بيانات الصفحة (مش ملفات جاهزة مرفوعة)، وتنزل على جهاز المستخدم مباشرة.

---

## 4. إزاي البيانات بتتحفظ (Data & Storage)

الموقع مبيستخدمش أي قاعدة بيانات أو سيرفر. كل حاجة (تشييكات، ملاحظات، محفوظات، اختيار الوضع الفاتح/الداكن) بتتحفظ في **localStorage** بتاع متصفح المستخدم، تحت مفتاح واحد اسمه `dpg_state_v1`.

**يعني ايه ده عمليًا؟**
- البيانات مربوطة بالمتصفح والجهاز اللي المستخدم فاتح منه الموقع — لو فتح الموقع من متصفح تاني أو جهاز تاني، مش هيلاقي نفس البيانات.
- مفيش أي بيانات بترجع لأي سيرفر أو بتتشارك مع حد تاني — كله محلي 100%.
- لو حبيت تمسح كل حاجة وترجّع الصفحة لأول مرة، افتح Developer Tools في المتصفح → Application → Local Storage → امسح المفتاح `dpg_state_v1`.

---

## 5. إزاي تعدّل المحتوى

كل المحتوى (المنتجات، الأدوات، الـ Roadmap، الـ Checklist) متعرّف كـ arrays في أول `js/app.js`:

```js
const PRODUCTS = [ {id:'saas', title:'...', tag:'...', icon:'...'}, ... ];
const RESOURCES = [ ... ];
const ROADMAP = [ ... ];
const CHECKLIST = [ ... ];
```

- عايز تضيف نوع منتج جديد؟ ضيف عنصر جديد في `PRODUCTS` بنفس الشكل (id فريد + عنوان + تصنيف + أيقونة من [Bootstrap Icons](https://icons.getbootstrap.com/)).
- عايز تغيّر نص أي مرحلة في الـ Roadmap؟ عدّل الـ `desc` بتاعها في `ROADMAP`.
- الصفحة بتعيد بناء نفسها تلقائيًا من الـ arrays دي (دالة `renderAll()`)، فمش محتاج تلمس أي HTML.

---

## 6. حدود بسيطة (Known Limitations)

- البحث بسيط (مطابقة نص جزئية) — مش بيفهم مرادفات أو أخطاء إملائية.
- التحميلات ملفات نصية `.txt` بسيطة، مش PDF منسّق.
- البيانات محلية للمتصفح فقط، زي ما اتشرح فوق — لو حابب مزامنة بين الأجهزة محتاج backend حقيقي (خارج نطاق النسخة دي).
